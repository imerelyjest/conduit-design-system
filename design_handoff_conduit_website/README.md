# Handoff — Conduit Systems website

This bundle is the **design reference** for the Conduit Systems marketing website. It is what your developer (or Claude Code) should rebuild against. Two pages are designed end-to-end:

1. **Landing page** — `site/index.html`
   Hero, Three-pillar offerings (CRM / Web app / Conduit Sites), integrations marquee, recent work / case studies, CTA break, footer.
2. **Conduit Sites page** — `site/Conduit Sites.html`
   Sites hero, "From kickoff to indexed" pipeline animation embed, recent launches showcase (with iframe previews + fullscreen overlay), pricing (Starter / Pro / Business), CTA break, footer.

Both pages share a top nav, footer, type/color tokens, animation library, and the Calendly popup integration.

---

## ⚠️ Read this first — what these files are (and aren't)

The HTML files in this bundle are **design references**, not production code to ship as-is.

- They are the canonical **look, copy, layout, motion, and component vocabulary**.
- They were built as a static reference against the Conduit design system (tokens, fonts, components).
- They are **not** wired to a CMS, a real form backend, analytics, routing, or a build pipeline.

Your developer's job is to **recreate these designs in the target codebase** — pixel-perfect — using whatever framework and patterns are right for the project (Next.js, Astro, plain HTML, etc.). If there is no codebase yet, pick the most appropriate framework and scaffold it.

**Fidelity:** High-fidelity. Final colors, typography, spacing, motion, and copy. Treat the visuals and copy as locked unless explicitly changed in conversation.

## Design tokens (already in `tokens/` and `colors_and_type.css`)

**Colors (5 anchors, 60/30/10/<5).**
- `--color-bg / --bg-page`: `#FBFBFE` (background, 60%)
- `--color-secondary / --fg-2`: `#212121` (secondary, 30%)
- `--color-primary`: `#FF4529` (primary, 10%) · pressed `#D43820`
- `--color-accent`: `#B7C8D7` (accent trim, <5%, used at 40% alpha for borders)
- `--color-fg / --fg-1`: `#040316` (text)

**Type.**
- Display + wordmark: **Plus Jakarta Sans 700**
- Body + UI: **DM Sans 400 / 500**
- Mono labels (kickers, meta): a monospace stack — letter-spacing `+0.18em`, UPPERCASE
- Scale: perfect fourth (1.333) anchored at 16px. H1/H2 fluid `clamp()`; H3–H5 fixed.
- Tracking: H1 `-0.02em`, H2 `-0.015em`, default elsewhere

**Spacing.** 4px base unit. `--section-py-desktop: 6rem`, `--section-py-hero: 11rem`. Container `1280px`, narrow `1024px`, body measure `62ch`.

**Radii.** Pills/buttons/nav `--radius-pill`. Cards `24px` (`--radius-xl`); featured card `32px` (`--radius-2xl`). Inputs `12px` (`--radius-md`). No square corners.

**Borders.** Hairline only. Subtle `rgba(183, 200, 215, 0.4)`. Active `2px solid var(--color-primary)` (with padding reduced 1px to keep dimensions stable).

**Shadows.** `--shadow-1` default card · `--shadow-3` hover lift · `--shadow-4` modal · `--shadow-primary-glow[-strong]` for primary CTAs and brand mark.

**Motion.** **One easing rules everything**: `cubic-bezier(0.2, 0.7, 0.2, 1)` (out fast, settle in slow).
Durations: `100ms` instant (focus) · `200ms` fast (button) · `240ms` base · `320ms` card reveal · `700ms` scene shift.
Hover = **brighten** (never darken). Buttons `transform: scale(1.02)` + stronger glow. Cards lift 2px and shift border to primary.
**One ambient loop per section** — never stack pulse + marquee + glass-drift in one viewport.
Every keyframe respects `prefers-reduced-motion`.

The full system is in `DESIGN_SYSTEM_REFERENCE.md` — read it before making any judgement calls.

## Interactions & behavior

- **Calendly popup.** Every element with `data-calendly` triggers `Calendly.initPopupWidget(...)`. Calendar URL: `https://calendly.com/colin-conduitsystems/30min`. There's also a floating "Book a Call" badge widget initialised on `load`. Inline scripts at the bottom of both pages.
- **Pulse rings (landing hero).** SVG dots generated in JS (`buildPulse()`), each ring offset in delay so motion radiates outward. Animates `transform` + `opacity` only.
- **Pipeline animation embed (Sites page).** Lazy-loaded via `IntersectionObserver` with a 1200ms timeout fallback; unloads when scrolled out.
- **Showcase iframe scaling.** `RENDER_W = 1440`. Iframes render at full width and are CSS-scaled down to fit the card. Recalculated on `load`, `resize`, and via `ResizeObserver`.
- **Fullscreen overlay.** Opens on `[data-embed-card]` click. Backdrop click + close button + Esc all close. While open, `document.body.style.overflow = 'hidden'` and the iframe is reset to `about:blank` 220ms after close so the fade plays out.
- **Marquee.** Two duplicated tracks, one `aria-hidden="true"`, animated via CSS. Fade-mask on track edges.

Forms / backend: **none in this bundle.** Wire the CTAs to whatever you ship with. If you replace Calendly, keep the brand color `#FF4529` on the popup.

1. **Download** this folder (the chat above this README has a download card — or grab the project zip).
2. **In your terminal**, drop the unzipped folder somewhere your repo can see it. A common pattern:
   ```
   your-repo/
   ├── design/                          ← put the unzipped bundle here
   │   └── design_handoff_conduit_website/
   ├── src/ ...
   └── ...
   ```
3. **Open Claude Code** in your repo root and start a new session. If your existing site is in the same repo and you want to overwrite it, **first commit or branch** the current state so you can roll back:
   ```bash
   git checkout -b conduit-redesign
   git add -A && git commit -m "snapshot before redesign"
   ```
4. **Prompt Claude Code** something like this:

   > Read `design/design_handoff_conduit_website/README.md` and `DESIGN_SYSTEM_REFERENCE.md` end-to-end before doing anything. The HTML files in `site/` are the **design reference** — recreate them in this codebase using our existing patterns (or set up a new <Next.js / Astro / etc.> project if there isn't one yet). Match the design pixel-perfectly. Use the tokens in `colors_and_type.css` and `tokens/`, the fonts in `fonts/`, the assets in `assets/`, and the motion library in `animations/`. Don't copy the static HTML wholesale — port it into proper components.
   >
   > Start by listing the components you'll create and the file structure you'll use. Wait for me to approve before you start writing code.

5. **Iterate.** Have Claude Code build one section at a time, screenshot it, and compare against `site/index.html` and `site/Conduit Sites.html` open side-by-side in a browser.

### What to tell Claude Code to **not** do

- Don't re-invent the color palette, type scale, spacing, or motion. Use the tokens.
- Don't darken on hover — Conduit always brightens.
- Don't use emoji, exclamation points, or enterprise jargon copy.
- Don't add gradient page backgrounds; backgrounds stay solid `--bg-page`.
- Don't ship the `tweaks*.jsx` files.
- Don't substitute fonts — they're self-hosted in `fonts/` for a reason.
